How this works

Data source: LLM Overwatch runs continuous probes from their own servers and publishes a free public JSON feed. This page reads that feed every ~5 minutes and writes data.js locally.

Claude Fable 5: Overwatch does not track Fable. Its numbers come from Artificial Analysis as a 72-hour median, refreshed ~every 3 hours (not live). Fable runs adaptive max-effort reasoning, so that median (~2 min end-to-end) includes thinking time — it is not comparable to the dial's near-real-time response times, and is shown in its own card rather than the dial.

What is measured: Response time in milliseconds, not answer quality. Measurements come from LLM Overwatch's servers to Anthropic's API — not from your machine. A fast response can still be wrong; a slow one can still be correct.

Dial: Each arc segment is one hour of the day. Brighter ice-cyan = faster (lower ms). Amber = unusually slow. The now-hand sweeps in real time, confined to the arc band.

Timezone: LLM Overwatch publishes hours in UTC; this page converts them to your local timezone () so the dial matches your day. Conversion uses your device's current UTC offset, so it follows daylight saving automatically.

Status badge: When available, pulled from the official status.claude.com API (summary.json). If that feed is unreachable, falls back to LLM Overwatch's measured status — labeled "(measured)".

Composite latency (the center number): A recency-weighted blend of Overwatch's own last-15-min, last-hour, and all-time averages — giving more weight to fresher data. This is a smarter read of one source's own numbers, not aggregated across providers. Deviation shown vs. the all-time baseline.

Confidence chip: HIGH/MED/LOW signal next to the number. It drops to MED or LOW when last-15-min samples are thin, data is stale (>20 min old), or the hourly average is missing. Hover for the reason.

Reconciliation strip: Compares the official status indicator against measured latency. "Operational, but latency N% above normal" means the status page shows green while measured response times are elevated — a common gap.

Stale data: If the feed hasn't updated in over 60 minutes, a ribbon appears above the dial. The last known data is still shown but greyed out.
